## Abstract

We present a framework for terrain generation using concepts inspired by hydrology, where rivers are used as modeling elements. The framework uses rivers as structural elements to create a hierarchical drainage network. The network is then analyzed to construct watersheds and characterize different types and trajectories of rivers. The terrain is finally generated by combining procedural terrain primitives and river patches with blending and carving operators. Our approach avoids the computational cost of simulating the physical processes underlying terrain formation, while generating realistic, complex terrain featuring river networks, valleys, hills, and mountains. The method enables users to sketch river networks interactively, generating rivers at a variety of spatial scales, and to simulate them in real time.

## 关键方程 / 核心结论

- TODO (Sprint 1A first read)
- Key method: use river network as terrain skeleton → watershed analysis → terrain generation by blending procedural patches around river paths
- Alternative to physics-based simulation: drainage-first terrain generation (vs. our physics-based approach of uplift → erosion → river extraction)
- Carving operators for valley generation — useful for Sprint 1A's river channel rendering pass

## 值得警惕的点

- TODO (Sprint 1A)
- Génevaux 2013 is CG-focused, not physically based — the terrain it generates is visually plausible but not geomorphically consistent. Use it as a rendering/visualization reference, not as a geomorphology reference
- The "drainage network" concept is the same as our flow accumulation graph but constructed artist-driven rather than physically derived
